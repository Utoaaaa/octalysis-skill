# Feedback Loop Design

## R — 原文 (Reading)

> "不要只停在這裡。請再做一次。"
>
> — 《遊戲化實戰全書》ch.17

## I — 方法論骨架 (Interpretation)

A behavior loop is stronger than a one-time reward path.

The loop asks what happens after the target behavior: What feedback confirms value? What asset, ability, relationship, insight, or status accumulates? How does that make the next action more likely?

A reward is weak if it ends the interaction. A reward is strong if it becomes an accelerator for the next behavior.

The loop must avoid dead ends. If users complete an action and nothing carries forward, the system is a line, not a loop.

## A1 — 書中的應用 (Past Application)

### 案例 1: Octalysis Prime concept

- **問題**: Learning content needed repeated engagement and deeper membership behavior.
- **方法論的使用**: Desired actions produce points, expertise, unlocks, social roles, and future accelerators.
- **結論**: Rewards become useful because they improve future participation.
- **結果**: The source uses this to demonstrate from-zero loop design.

### 案例 2: P&G / NAVA Orbico

- **問題**: Sales, training, and collaboration needed ongoing behavior.
- **方法論的使用**: KPI-linked actions advanced personal and shared worlds, creating fast feedback and team progress.
- **結論**: Work behavior should feed back into both individual and group loops.
- **結果**: Source cites strong participation and business improvement.

## A2 — 觸發場景 (Future Trigger)

### 用户會在什麼情境下需要這個 skill?

1. Users take an action once but do not return.
2. Rewards are claimed but do not reinforce the target behavior.
3. A habit, course, workflow, or community needs a repeatable loop.
4. The user wants to design streaks, points, unlocks, or rewards safely.
5. A system has dead ends after completion.

### 語言信號

- "怎麼讓人下次還會回來?"

## E — 可執行步驟 (Execution)

1. **Define the repeatable action**
   - Name the smallest target behavior that should repeat.
   - Completion standard: the action can be observed and counted.

2. **Map the current loop**
   - Trigger -> action -> feedback -> reward/asset -> re-entry.
   - Completion standard: every link is explicit; missing links are marked.

3. **Classify the feedback**
   - Determine whether feedback creates progress, meaning, ownership, social value, curiosity, scarcity, or loss protection.
   - Completion standard: feedback is tied to a core drive.

4. **Convert rewards into accelerators**
   - Redesign rewards so they make the next action easier, richer, more social, more meaningful, or more capable.
   - Completion standard: each reward has a re-entry function.

5. **Remove dead ends**
   - Add next action, new role, unlock path, social contribution, or accumulated asset after completion.
   - Completion standard: the loop has a clear next reason to return.

6. **Check risks**
   - Review streaks, random rewards, sunk cost, and social pressure.
   - Completion standard: risky loops have guardrails and exit paths.

## B — 邊界 (Boundary)

### 不要在以下情況使用此 skill

- The desired behavior is intentionally one-time.
- Repetition would be harmful, unsafe, or unwanted.
- The system lacks a clear target behavior.

### 作者在書中警告的失敗模式

- Reward schedules and economies need careful design; adding all ideas at once shows lack of focus.
- Loops can create unhealthy compulsion if CD6/CD7/CD8 dominate.
- The best loop still needs user value, not just re-entry pressure.

### 作者的盲點 / 時代局限

- Many loop examples come from digital products; for physical, workplace, or agent workflows, feedback may be artifacts, reviews, dashboards, or saved decisions.

### 容易混淆的鄰近方法論

- A checklist ends after completion.
- A behavior loop makes completion feed the next action.
