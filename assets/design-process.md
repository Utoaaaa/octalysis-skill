# Universal Motivation Design Process

Use this process when a request asks to diagnose, redesign, or create motivation for a behavior.

## 1. Frame The Behavior

- Define the success metric or target outcome.
- Define the motivated actor.
- Define the target behavior.
- Define current stage: Discovery, Onboarding, Scaffolding, Endgame.
- Identify whether the main problem is motivation, ability, trust, access, or friction.

If the problem is mostly ability/access/trust, fix that before motivation design.

## 2. Build The Strategy Dashboard

Minimum viable information:

1. Ranked success metrics.
2. Motivation segments.
3. Target behavior path.
4. Feedback mechanisms and media.
5. Incentives/rewards.
6. Win states.

If success metrics conflict, state the tradeoff before designing.

## 3. Diagnose Current Motivation

Score CD1-CD8 qualitatively:

- strong,
- weak,
- missing,
- overused,
- misused.

Also classify:

- left-brain/extrinsic vs right-brain/intrinsic,
- white-hat vs black-hat,
- implicit vs explicit mode.

## 4. Select Intervention Direction

Pick 1-3 primary intervention drives.

Do not try to maximize all eight. A good design has a coherent motivation shape.

For each selected drive:

- specify the actor need,
- intervention pattern,
- feedback,
- trigger,
- risk,
- measurement signal.

## 5. Build The Behavior Loop

For repeated behavior:

1. Trigger.
2. Target action.
3. Feedback/win state.
4. Reward/asset/capability.
5. Re-entry reason.
6. Next action.

Reject rewards that do not strengthen re-entry unless the behavior is intentionally one-time.

## 6. Run Ethics Review

Mandatory if using CD6/CD7/CD8, social pressure, monetary rewards, streaks, random rewards, public comparison, lock-in, or scarcity.

## 7. Output

Recommended output format:

- Current motivation diagnosis.
- Key missing/overused drives.
- Primary design strategy.
- Concrete interventions.
- Behavior loop.
- Risks and guardrails.
- Measurement plan.

## 8. Measurement

Measure not only target metrics but also:

- voluntary return,
- completion quality,
- user sentiment after action,
- regret or complaint signals,
- opt-out/exit rate,
- long-term fatigue,
- trust.
