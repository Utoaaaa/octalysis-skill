# Diagnostic Rules

Use these rules whenever diagnosing or designing motivation. They are phrased as action rules for an agent.

## Rule 1 — Start From Target Behavior

If the request starts with "make this engaging," "gamify this," or "add rewards," first define:

1. Success metric / target outcome.
2. Motivated actor.
3. Target behavior.
4. Current journey stage.
5. Current blocker.

Do not propose mechanics until those are known or explicitly assumed.

## Rule 2 — Inspect All Eight Drives

For the target behavior, ask:

- CD1: Is there meaning, identity, contribution, or calling?
- CD2: Is there visible progress, mastery, challenge, and completion?
- CD3: Is there meaningful choice, experimentation, expression, and feedback?
- CD4: Does action create ownership, assets, personalization, or stewardship?
- CD5: Is there belonging, recognition, reciprocity, trust, or social proof?
- CD6: Is there a meaningful window, constraint, sequence, or unlock?
- CD7: Is there curiosity, discovery, surprise, or unresolved information?
- CD8: Is there real loss, regret, risk, or status quo inertia?

Then decide which drives are missing, overused, or misaligned.

## Rule 3 — Underuse Is Not Laziness

When users abandon or delay:

- Locate the exact drop-off behavior.
- Identify which drive was spent and not replenished.
- Check whether completion has a real win state.
- Add drive-specific feedback at the drop-off point.

Do not default to "users are lazy" or "send more reminders."

## Rule 4 — Overuse Shows Up As Narrow Motivation

If metrics rise while complaints, churn, shame, or reward dependency rise:

- Check for excessive CD2 score/status chasing.
- Check for excessive CD4 lock-in or sunk cost.
- Check for excessive CD6/CD7/CD8 urgency, randomness, or loss.
- Rebalance with CD1, CD3, and CD5.

## Rule 5 — Rewards Are Entry And Justification, Not The Whole Design

Use extrinsic rewards when:

- the task has little intrinsic appeal,
- the actor needs a reason to try,
- the reward confirms real progress,
- the reward makes the next action more useful.

Avoid extrinsic rewards when:

- the behavior is already intrinsically enjoyable,
- reward quality will shrink over time,
- users will optimize for the reward rather than the behavior.

## Rule 6 — Black-Hat Requires Fit, Consent, And Time Bounds

Any CD6/CD7/CD8 design must answer:

- Is the target behavior something the actor would endorse on reflection?
- Is the pressure truthful and proportionate?
- Can the actor pause, recover, or exit?
- What white-hat feedback follows the urgent action?
- What happens after repeated exposure?

## Rule 7 — Prioritize Metrics Before Interface Or Copy

If there are multiple desired outcomes, rank them.

The primary success metric determines:

- the main call to action,
- the target behavior sequence,
- the strongest trigger,
- the win state,
- which tradeoffs are acceptable.

If the priority changes, redesign the motivation logic.

## Rule 8 — Segment By Motivation

Do not assume one design motivates everyone.

For each segment:

- What same target behavior must they perform?
- Which drives are naturally strongest?
- Which drives are likely resisted?
- Does this group need a different trigger, feedback, or win state?

If groups have different target behaviors, they are not just segments; they are different games.

## Rule 9 — Loyalty Is Relationship, Not Reward Rate

If users only stay for discounts, points, or cashback, call it a mercenary program.

To design loyalty, add:

- identity,
- trust,
- status,
- access,
- community,
- history,
- meaningful surprise,
- useful power.

## Rule 10 — Rewards Should Re-Enter The Loop

A good reward should make the next target behavior:

- easier,
- richer,
- more meaningful,
- more social,
- more capable,
- more personalized,
- or more worth returning to.

If the reward only creates a pleasant endpoint, it is an exit reward.

## Rule 11 — Match Reward Timing To Behavior Cost

- Use fixed rewards when the behavior is costly, high-friction, or requires clarity.
- Use variable rewards only for short, low-cost, bounded actions.
- Use Easter eggs after a win state, not as the main reason to act.
- Use social treasure when giving to others reinforces relationship.
- Use collections when pacing and ownership matter.

## Rule 12 — Avoid Treating Cases As Templates

For every case transfer, extract:

- target behavior,
- audience,
- missing drives,
- intervention drives,
- feedback medium,
- why it worked,
- non-transferable context,
- misuse warning.

If only the surface mechanic transfers, reject the analogy.
